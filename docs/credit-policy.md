# OPOD 크레딧 / 요금 정책

이 문서는 크레딧 가격과 차감 정책의 원본이다. 구현 상수는
src/domain/credits/credit-pricing.ts와 일치해야 한다.

## 핵심 결정

| 항목 | 정책 |
|---|---|
| 과금 모델 | 실행 전에 가격을 알 수 있는 액션당 정액 종량제 |
| 재화 단위 | 웹 결제 기준 1크레딧 = ₩10 |
| 기본 채팅 | chat_reply 1회에 2크레딧 |
| 무료 지급 | 가입 100크레딧, 출석 10크레딧, 무상 크레딧 30일 유효 |
| 차감 | reserve → capture / release |

토큰 사용량이나 대화 길이에 따라 사후 가격을 바꾸지 않는다. 모델과
컨텍스트 비용은 서버의 입력 상한과 모델 선택으로 통제하며, 고급 기능이
필요하면 실행 전에 가격이 보이는 별도 액션으로 제공한다.

## 크레딧 종류

- 유상 크레딧은 결제로 취득하며 유효기간이 없다.
- 무상 크레딧은 가입, 출석, 이벤트, CS 보상으로 지급하며 30일 동안
  유효하다.
- 차감 순서는 소멸이 가까운 무상 크레딧, 나머지 무상 크레딧, 유상
  크레딧 순서다.

## 액션 가격

| 액션 | 가격 |
|---|---:|
| chat_reply | 2크레딧 |

새 액션은 추정 원가와 운영 마진을 검토한 후 이 표와
credit-pricing.ts에 함께 추가한다.

## 충전 패키지

| 패키지 | 가격 | 크레딧 |
|---|---:|---:|
| credits_500 | ₩4,900 | 500 |
| credits_1050 | ₩9,900 | 1,050 |
| credits_3300 | ₩29,000 | 3,300 |
| credits_5750 | ₩49,000 | 5,750 |

유상 크레딧은 유효기간이 없다. 앱 결제 가격은 마켓 수수료를 반영할 수
있지만 지급 크레딧과 최종 가격은 결제 전에 표시해야 한다.

## 무료 크레딧

- 가입 완료 시 100크레딧을 한 번 지급한다.
- KST 날짜 기준 하루 한 번 출석하면 10크레딧을 지급한다.
- 월 누적 출석 7일, 14일, 30일에 각각 20, 30, 50크레딧을 추가 지급한다.
- 모든 무상 지급분은 지급일부터 30일 동안 유효하다.

## 예약과 확정

    액션 요청
      → reserve: 가용 잔액 확인과 예약
      → 성공: capture 후 원장에 debit 기록
      → 실패·timeout: release, 원장 기록 없음

- 가용 잔액은 원장 잔액에서 활성 예약을 뺀 값이다.
- 잔액 확인과 예약 생성은 사용자 단위로 직렬화된 트랜잭션에서 수행한다.
- 예약 reference는 unique하며 capture와 release는 멱등적이어야 한다.
- 기본 예약 TTL은 5분이다.
- 잔액이 부족하면 액션 실행이나 메시지 저장 전에
  INSUFFICIENT_CREDITS로 거절한다.

## 운영 정책

- 결제 취소 시 해당 구매로 지급된 크레딧을 회수한다.
- 서비스 실패가 capture된 경우 확인 후 보상 grant를 지급한다.
- 관리자 수동 지급은 CS 보상과 이벤트에만 사용하며 사유를 기록한다.
- 단가, 패키지, 무료 지급량 변경은 최소 7일 전에 공지하고 문서와 구현
  상수를 함께 갱신한다.
