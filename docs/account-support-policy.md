# OPOD 계정 관리 / 고객지원 정책

범위: 비밀번호 변경, 회원탈퇴, FAQ, 공지사항, 1:1 문의.
상세 HTTP 계약은 docs/api/auth.md와 docs/api/support.md를 따른다.

## 핵심 결정

| 항목 | 정책 |
|---|---|
| 비밀번호 변경 | 현재 비밀번호를 확인하고 모든 refresh token을 폐기한 뒤 새 token pair를 발급한다. |
| 회원탈퇴 | 유예 없이 즉시 익명화하며 복구할 수 없다. |
| 재가입 | 즉시 허용하되 탈퇴 후 30일 동안 가입 보너스를 다시 지급하지 않는다. |
| 이메일 보관 | 원문이나 암호문을 보관하지 않고 HMAC hash만 30일 보관한다. |
| FAQ·공지 | 비로그인 공개 조회이며 작성과 발행은 opod-admin이 담당한다. |
| 1:1 문의 | 로그인한 사용자의 단건 문의와 단건 답변만 지원한다. |

## 비밀번호 변경

- 새 비밀번호는 가입과 동일한 8~128자 규칙을 사용하며 현재 비밀번호와
  달라야 한다.
- 변경 성공 시 새 salt와 hash를 저장하고 기존 refresh token을 모두
  삭제한다.
- 새 token pair를 발급해 현재 기기만 로그인 상태를 유지한다.
- 성공 이벤트 auth.password_changed를 기록한다.

## 회원탈퇴

- 비밀번호 재확인이 필요하다.
- users 행은 원장과 결제 기록의 참조 무결성을 위해 유지하되 이메일,
  비밀번호 정보와 표시 이름을 익명화하고 deletedAt을 기록한다.
- refresh token, 대화, 알림, 팔로우, 해시태그 선호는 삭제한다.
- 크레딧·결제 원장, 신고, 문의, 비식별 이벤트는 법적·운영상 필요한 기간
  유지한다.
- 잔여 크레딧은 소멸한다. 환불이 필요하면 탈퇴 전에 처리해야 한다.
- 모든 인증 조회는 deletedAt IS NULL을 요구한다.

재가입 보너스 반복 수령을 막기 위해 탈퇴 시
HMAC-SHA256(email, AUTH_EMAIL_HASH_PEPPER)을 기록한다. 가입 시 최근 30일
안에 같은 hash가 있으면 가입은 허용하되 보너스를 지급하지 않는다. 30일이
지난 hash는 파기한다.

## FAQ

- 발행된 항목만 공개한다.
- 정렬은 sortOrder 오름차순, createdAt 내림차순이다.
- 카테고리는 운영 중 추가할 수 있는 문자열이다.
- 데이터 규모가 작으므로 최대 200건을 반환하고 페이지네이션하지 않는다.

## 공지사항

- publishedAt이 현재보다 이전인 항목만 공개한다.
- 고정 공지는 첫 페이지의 별도 pinned 목록으로 반환한다.
- 일반 목록은 publishedAt 내림차순 커서 페이지네이션을 사용한다.
- 목록은 본문을 제외하고 상세 조회가 본문을 제공한다.

## 1:1 문의

- 카테고리와 최대 2,000자의 본문을 받으며 첨부파일과 스레드는 지원하지
  않는다.
- 상태는 submitted → answered만 사용한다.
- 사용자는 답변 전 문의만 삭제할 수 있다.
- KST 기준 사용자당 하루 10건으로 제한한다.
- 타인의 문의 존재 여부를 노출하지 않도록 소유하지 않은 문의는 404로
  응답한다.
- 답변 저장 시 같은 트랜잭션에서 inquiry.answered 알림을 만든다.

## 저장소 경계

| 기능 | opod-service-backend | opod-admin |
|---|---|---|
| 비밀번호 변경·탈퇴 | 전체 동작 | 탈퇴 통계 조회 |
| FAQ·공지 | schema와 공개 조회 | 작성·수정·발행·정렬 |
| 문의 | 접수·조회·삭제 | 답변과 답변 알림 |

## 알려진 제한

이메일 발송 인프라가 없어 이메일 인증과 비밀번호 재설정을 제공하지 않는다.
도입 전까지 비밀번호를 잊은 계정은 복구할 수 없으므로 FAQ에 별도 문의
채널을 안내한다.
