# 결제·환불 테스트 유스케이스

실제 결제금액의 승인, 환불, 차지백 또는 정산 상태에 영향을 주는
시나리오만 다룬다. 가입·출석·일반 무상 지급은 제외하되, 구매 연계
프로모션처럼 환불 회수액에 영향을 주는 경우는 포함한다.

## 공통 기준

- 사용자 귀책 환불은 구매별 최초 유상 크레딧의 50% 이상이 남았을 때
  잔여 유상 크레딧 전체를 환불한다.
- 사용자 귀책 환불 수수료는 환불 대상 금액의 5%이며 원 단위 미만은
  버린다.
- 회사 귀책, 중복 결제, 과오금, 법정 환불에는 50% 조건과 수수료를
  적용하지 않는다.
- 유상 크레딧은 가장 오래된 구매분부터 사용한다.
- 사용자 귀책, 앱마켓 강제 환불, 차지백에서는 구매 연계 프로모션을
  전량 회수하고 부족분은 유상 잔액의 음수로 반영한다.
- 회사 귀책 환불에서는 구매 연계 프로모션을 유지한다.
- 환불 요청 중인 크레딧은 잠가서 사용할 수 없게 한다.
- 결제사 성공 응답을 확인한 시점을 환불 완료로 본다.

계산식:

```text
환불 대상 금액
  = floor(실제 결제금액 × 환불 유상 크레딧 ÷ 최초 유상 크레딧)

수수료
  = floor(환불 대상 금액 × 5%)

최종 환불액
  = 환불 대상 금액 - 수수료
```

## 핵심 불변 조건

모든 테스트는 다음 조건도 함께 검증한다.

1. 동일 주문은 한 번만 크레딧을 지급한다.
2. 동일 환불은 한 번만 돈을 돌려주고 한 번만 크레딧을 회수한다.
3. 누적 환불액은 실제 결제액을 초과하지 않는다.
4. 결제 성공 없이 유상 크레딧이 순증하지 않는다.
5. 환불 성공 후 회수되지 않은 구매 크레딧이 남지 않는다. 단, 회사
   귀책 환불의 구매 연계 보너스는 예외다.
6. 환불 실패 시 사용자의 돈과 크레딧이 동시에 사라지지 않는다.
7. 사용자 귀책 환불 중인 크레딧은 사용과 환불에 중복 소비되지 않는다.
8. 음수 유상 잔액이 있는 동안 무상 잔액이 있어도 크레딧 서비스는
   실행되지 않는다.
